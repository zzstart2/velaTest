| 3.2 | 弹窗内容 | 当前绑定手机号（脱敏只读） + 短信验证码 + 新密码 + 确认新密码 + 取消 / 确认按钮 |

## 四、换绑手机（已登录态）

| # | 用例 | 预期结果 |
|---|---|---|
| 4.1 | 点击「换绑手机」 | 弹窗打开，进入 Step 1（校验旧手机），步骤指示器同步 |
| 4.2 | Step 1 — 旧手机短信验证 | 显示当前绑定号码（脱敏） + 验证码输入；验证码错误时提示并停留 |
| 4.3 | Step 1 校验通过 | 进入 Step 2，步骤指示器同步推进 |
| 4.4 | Step 2 — 新手机绑定 | 固定 +86 前缀 + 新手机号 + 新手机短信验证码 + 上一步 / 确认换绑按钮 |
| 4.5 | 点「上一步」 | 退回 Step 1，步骤指示器同步回退 |
| 4.6 | 新号码与原号码相同 | 后端拒绝，前端提示新号码与原号码相同 |
| 4.7 | 新号码已被另一账号绑定 | 后端拒绝，前端提示该号码已绑定其他账号 |
| 4.8 | 新号码为同号多账号占用 | 后端拒绝，前端提示该号码已绑定多个账号、无法绑定 |
| 4.9 | 新手机验证码错误 | 提示并保留弹窗 |
| 4.10 | 全部校验通过 | 进入 Step 3（完成），动态显示新号码脱敏，关闭弹窗后下次打开重置回 Step 1 |
| 4.11 | 各 Step「发送验证码」按钮 | 各自独立倒计时 |
| 4.12 | 三个控制台页面行为一致 | dashboard / api-keys / usage 三页弹窗内容、字段、交互完全相同 |
