# SenseNova 账号体系 PRD

> 版本：v1
> 时间：2026-04
> 范围：登录 / 注册 / 忘记密码 / 修改密码 / 换绑手机 / 多账号选择
> 关联原型：`login.html` · `register.html` · `console/{dashboard,api-keys,usage}.html`

---

## 1. 名词

| 名词 | 含义 |
|---|---|
| 用户名 | 账号唯一标识，登录凭据，注册时设置后不可修改 |
| 绑定手机号 | 账号下挂的手机号，用于手机号登录、忘记密码、换绑等 |
| 同号多账号 | 历史遗留：一个手机号曾被绑定到 ≥ 2 个账号；该号码不可再创建新账号 |
| 已校验手机号 | 通过 `/auth/sms/send` + `/auth/phone/verify` 拿到的、有时效的 phone token |

---

## 2. 字段约束

### 2.1 用户名
- 长度：6–24 位
- 字符集：英文字母（区分大小写）+ 数字（`[a-zA-Z0-9]`）
- 全局唯一
- 注册后不可修改
- 提示语："请设置用户名 / 6–24 个英文、数字字符，区分大小写"

### 2.2 密码
注册、忘记密码、修改密码、换绑后下发的新密码均遵循以下规则；前端实时勾选展示：
- 至少一个小写字母（a–z）
- 至少一个大写字母（A–Z）
- 至少一个数字（0–9）
- 至少一个特殊字符 `~!@#$%^&*?_+-.,;:`
- 长度 8–32 位

### 2.3 手机号
- 仅支持中国大陆 +86 手机号
- 仅用于手机号登录 / 注册一体入口、忘记密码、换绑手机
- 不可作为账号密码登录的标识符

### 2.4 短信验证码
- 6 位数字
- 单手机号 60 秒发送间隔
- 单次有效期 5 分钟
- 发送、校验接口不区分新老用户 / 旧手机 / 新手机

## 3. 登录流程

登录页有两个 Tab，**默认进入"手机号登录 / 注册"** Tab。

### 3.1 Tab A · 账号密码登录（仅登录）
1. 输入**用户名** + 密码
2. 点击「登录」
3. 后端校验 `username + password_hash`
   - 成功 → 颁发 session → 跳 `/console/dashboard`
   - 账号不存在 / 密码错误 → 字段下错误提示
   - 多次失败触发账号锁定（按现有风控策略）
4. 链接「忘记密码？用手机号重置 →」 → 切到"忘记密码"视图（见 §5）

> ⚠ 账号密码登录不接受手机号作为标识

### 3.2 Tab B · 手机号登录 / 注册一体入口
**Step 1：手机号 + 验证码**
1. 输入手机号 + 短信验证码
2. 后端校验通过后查询该号码绑定账号数：

| 账号数 | 行为 |
|---|---|
| 0 | 携带 `cc / phone / verified=1` 跳 `register.html`，进入注册流程，无需重发验证码 |
| 1 | 直接颁发该账号 session → 跳 `/console/dashboard` |
| ≥ 2 | 进入 Step 2：多账号选择 |

**Step 2：多账号选择（仅当账号数 ≥ 2）**
1. 展示该号码绑定的账号卡片列表（用户名 + 头像）
2. 用户点选某账号 → 颁发该账号 session → 跳 `/console/dashboard`
3. 该号码**禁止创建新账号**：注册接口若收到此号码的注册请求，返回 409
4. 该步骤无"返回上一步"入口（避免误退导致状态丢失）

## 4. 注册流程

只能由"手机号登录 / 注册"Tab 在号码"未注册"分支触发。

### 4.1 进入条件
- 必须携带 `?cc=&phone=&verified=1`（phone token 由 Step 1 颁发，5 分钟有效）
- 未携带或 token 失效 → 返回 `login.html`
- 号码已存在多账号绑定 → 注册接口返回 409，跳回 `login.html`

### 4.2 表单字段
顶部展示绿色已校验手机号横幅（脱敏 `+86 138****0000`），右侧"更换手机号"链接可返回 Step 1。

| 字段 | 约束 | 备注 |
|---|---|---|
| 用户名 | §2.1 | 设置后不可修改 |
| 设置密码 | §2.2 | 5 项规则实时勾选 |
| 确认密码 | 与"设置密码"完全一致 | 前端实时一致性提示 |
| 服务条款 + 隐私政策 | 必勾 | — |

### 4.3 提交
- 接口：`POST /auth/register { phone_token, username, password }`
- 后端再次校验 phone_token；账号唯一性；密码规则
- 成功 → 创建账号，默认绑定该手机号 → 自动颁发 session → 跳 `/console/dashboard`
- 失败：
  - 用户名已存在 → 字段下提示"该用户名已存在"
  - phone_token 失效 / 该号码已多账号 → 跳回 `login.html`

---

## 5. 忘记密码流程（未登录态）

入口：账号密码 Tab 的"忘记密码？用手机号重置 →"链接。

### 5.1 字段
- 用户名（要重置密码的目标账号）
- 该账号的绑定手机号
- 短信验证码（发到该手机号）
- 新密码（§2.2）
- 确认新密码

### 5.2 校验
- 前端：两次密码一致 + 满足密码规则
- 后端：账号存在 ∧ 手机号确为该账号绑定号码 ∧ 验证码有效

### 5.3 结果
- 成功 → 落库新 password_hash → 自动颁发 session → 跳 `/console/dashboard`
- 校验失败 → 字段下提示"用户名或绑定手机号不正确"
- 同号多账号场景：仍按 `username + phone` 双因子校验，不会因多账号产生歧义

页面顶部"← 返回登录"可回到登录 Tab。

---

## 6. 修改密码流程（已登录态）

入口：控制台右上角用户菜单「修改密码」。

### 6.1 弹窗字段
- 当前绑定手机号（只读展示）
- 短信验证码（发到当前绑定手机号）
- 新密码（§2.2）
- 确认新密码

### 6.2 校验与提交
- 前端：两次密码一致 + 满足密码规则
- 后端：当前 session 有效 ∧ 验证码有效
- 成功 → 落库新 password_hash → 弹窗关闭，提示"密码修改成功"

> 与"忘记密码"的差异：已登录态无需输入用户名，直接用绑定手机号校验即可。

---

## 7. 换绑手机流程（已登录态）

入口：控制台右上角用户菜单「换绑手机」（与"修改密码"同一菜单）。

### 7.1 三步式弹窗
**Step 1 · 校验旧手机**
- 展示当前绑定手机号（只读）
- 输入短信验证码（发到旧手机）
- 校验通过 → Step 2

**Step 2 · 绑定新手机**
- 输入新手机号（+86）
- 输入短信验证码（发到新手机）
- 校验通过 → 提交换绑

**Step 3 · 完成**
- 显示成功提示，含新号码脱敏展示
- 关闭弹窗

### 7.2 新号码冲突约束
| 场景 | 后端响应 | 前端提示 |
|---|---|---|
| 新号码已被同号多账号占用 | 409 | "该号码已绑定多个账号，无法绑定" |
| 新号码已被另一单账号绑定 | 409 | "该号码已绑定其他账号" |
| 新号码与原号码相同 | 400 | "新号码与原号码相同" |
| 验证码错误 | 401 | 字段下报错 |

### 7.3 生效
- 新号码即时生效，下次手机号登录、忘记密码均使用新号码
- 旧号码立即解绑，不再可登录该账号

## 8. 接口口径建议

| 接口 | 用途 | 入参要点 |
|---|---|---|
| `POST /auth/sms/send` | 发送验证码 | `{ cc, phone }`，60s 限流 |
| `POST /auth/phone/verify` | 校验验证码 | `{ cc, phone, code }` → 返回 `{ phone_token, accounts: [...] }` |
| `POST /auth/login` | 登录 | 二选一：① `{ username, password }` ② `{ phone_token, account_id }` |
| `POST /auth/register` | 注册 | `{ phone_token, username, password }`；号码已多账号则 409 |
| `POST /auth/password/reset` | 忘记密码 | `{ username, cc, phone, sms_code, new_password }` |
| `POST /account/password/change` | 已登录改密 | `{ sms_code, new_password }` |
| `POST /account/phone/rebind` | 换绑手机 | `{ old_sms_code, new_cc, new_phone, new_sms_code }` |

## 9. 同号多账号约束 · 总结

| 场景 | 行为 |
|---|---|
| 手机号登录 | 多账号选择步骤，用户挑选账号登录 |
| 注册新账号 | **禁止**，注册接口返回 409 |
| 忘记密码 | 允许，按 `username + phone` 双因子定位账号 |
| 换绑手机（旧→该号码） | 不涉及（旧号码不会主动选成已多账号号码） |
| 换绑手机（新=该号码） | **禁止**，409 提示"该号码已绑定多个账号，无法绑定" |

---

## 10. 一些隐含约束 / 待确认项

- 用户名是否允许中文 / 下划线 / 短横线？当前规则**禁止**，仅英文 + 数字
- 多账号选择列表是否需要展示账号副信息（如最近登录时间、账号类型）？当前**不展示**，避免信息泄露
- 修改密码后是否强制重新登录？建议**保持当前 session**，但失效其他设备 session
- 换绑手机后是否发送通知短信到旧号码？建议**发送**，便于风险感知（待确认）
- Phone token 有效期：建议 5 分钟，与短信验证码一致
- 用户名锁定策略：当前不区分是哪个 Tab 的失败次数，按用户名维度统一计数
