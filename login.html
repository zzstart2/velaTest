A 账号密码登录

  1. 用户输入账号（企业标识 / 用户名） + 密码（不接受手机号）
  2. 后端校验 username + password_hash
  3. 校验通过 → 颁发 session/token → 跳转 /console/dashboard
  4. 失败：返回错误码（账号不存在 / 密码错误 / 锁定）
  5. "忘记密码"入口 → 切到 D 忘记密码 视图

B 手机号登录 / 注册一体入口

Step 1：输入手机号 + 短信验证码 → 后端校验通过后，按手机号匹配账号数量分支：

分支 1 · 0 个账号（未注册）
→ 携带 cc / phone / verified=1 跳 register.html，手机号已校验，无需重发验证码
分支 2 · 1 个账号（已注册）
→ 直接颁发 session → 跳 /console/dashboard
分支 3 · ≥ 2 个账号（历史多账号）
→ 返回账号列表，进入 Step 2 选择账号
→ 用户选定账号 → 颁发该账号 session → 跳 dashboard
→ 该手机号禁止创建新账号（注册接口需对该号码返回 409）

C 接口口径建议

  • POST /auth/sms/send — 发送验证码（60s 限流，不区分新老用户 / 旧手机 / 新手机）
  • POST /auth/phone/verify — 校验验证码，返回 { accounts: [...] } 数组（0 / 1 / N）
  • POST /auth/login — 入参 account_id + 已校验的 phone token，颁发 session
  • POST /auth/register — 仅当 verify 返回的 accounts 数组为空时可调用
  • POST /auth/password/reset — 忘记密码用：入参 { account, phone, sms_code, new_password }
  • POST /account/phone/rebind — 已登录态换绑：入参 { old_sms_code, new_phone, new_sms_code }

D 忘记密码（未登录态）

入口：账号密码 Tab 上的"忘记密码？"链接 → 切换到忘记密码视图

  1. 用户输入账号 + 该账号绑定的手机号 + 短信验证码 + 新密码 + 确认密码
  2. 前端校验：两次密码一致、密码强度达标、账号格式合法
  3. 后端校验：账号存在 ∧ 手机号确为该账号绑定号码 ∧ 验证码有效
校验失败 → 返回 401/404，前端字段下提示"账号或绑定手机号不正确"
校验成功 → 落库新 password_hash → 自动颁发 session → 跳 dashboard
  • 注意：若账号绑定的是同号多账号中的一个，校验时仍按 account + phone 双因子；不会因多账号导致歧义
  • 已登录态走"修改密码"弹窗（用绑定手机号即可，无需再输账号）

E 换绑手机（已登录态）

入口：控制台用户菜单"换绑手机"

  1. Step 1：校验旧手机短信验证码（由后端比对当前账号绑定号码）
  2. Step 2：输入新手机号 + 新手机短信验证码
  3. Step 3：完成提示，新号码即时生效
新号码冲突校验：若新号码已被同号多账号占用，返回 409，前端提示"该号码已绑定多个账号，无法绑定"
新号码已被单账号绑定：返回 409，提示"该号码已绑定其他账号"
